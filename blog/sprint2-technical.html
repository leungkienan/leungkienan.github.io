Classes vs ID's

What is a Class?

Classes are used to identify multiple elements. These are notated with a "." as seen below

When and why do we use classes?

We want to use classes when you want to style many elements on the page. So if you know you will use this element with the same style, we will tag the element with the class tag. Classes are also great for giving a group of elements specific attributes, so we can have headers that have the same features, but some may have different colours or different font styles. One would use a class to style elements that they want consistent across their code, so things like multiple text-boxes on a page may have the same attributes like colour, sizing or borders, etc.

Summary

Classes give you plenty of freedom to style and manipulate many elements in your code, so we can give elements in our html file some flair and unique styling.

What is an ID?

An ID is used to identify a single element. These are annotated as a '#'

When and why do we use ID's?

ID's are specifically used for only one element, so we should use ID for elements that have only one instance of itself per page. So we won't create multiple objects with an ID tag, as these will clash with each other once we dip into JavaScript.

For instance, if we opened up my page's elements in the browser, I have the main content of the page to be classified as an ID (highlighted green), as I want this element to remain consistent across all of my pages.
