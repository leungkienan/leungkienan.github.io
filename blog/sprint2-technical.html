Classes vs ID's

Best Practices for Classes and IDs

Think of a storage warehouse, full of different boxes with varying contents inside. With that in mind, we can assign the warehouse with an id element

So our warehouse could be setup so that id = warehouse-storage

We know that there are boxes inside the warehouse, so we can tag these as a class element

class = box

If we were to make a div with these elements, it would look like the following:

We can see that there is a warehouse with the id "warehouse-storage" setup, with a box with the class "box" setup.

Why do we use different tags?

The warehouse has its own specific ID that we can identify, which has its own unique aspects to it. So if we wanted to do something to the warehouse, like expand it or knock down a wall, we can locate the specific warehouse we want to modify and make changes to that warehouse.

So with an ID, we can target unique elements and give them special functions or styles. We also use ID's to call certain elements for a specific purpose with JavaScript, but if there are multiple elements with the same ID, our scripts won't know what element to run through and will show an error.

Classes do not share this restriction, as we can have multiple elements which share the same attributes and characteristics with one another. So our boxes can have the same properties, such as size and shape.

Unlike IDs, an element can have multiple classes to distinguish one another. So back to our box analogy, we can have a bunch of boxes with televisions in them, so we can give them a "tv" class to categorise them.

As seen above, we annotate the class with a period, or a '.', whereas the id is annotated with a '#'.

We can also weave id's with classes, as the element may require some JavaScript properties to make a particular action occur too.

Summary

Using these different classifications allows us to get more narrowed and specific styling of particular objects in our webpages

Going forward, classes are a great way to make your elements stand out and play around with. ID's are great once JavaScript kicks in, as these elements will be using functions and all sorts of commands to make the user experience to the best it can be, but for now, they are a good for separating portions of your webpage into blocks.
